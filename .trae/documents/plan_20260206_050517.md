# Plan de Implementación para Continuous Deployment (CD)

Para automatizar el release y publicación a npm usando GitHub Actions, implementaremos un flujo de "Direct Release" en la rama `master`. Esto elimina la necesidad de intervención manual (OTP) y de crear PRs intermedias para el changelog, alineándose con las prácticas de CD.

## 1. Cambios de Configuración (`nx.json`)
Necesitamos habilitar la creación automática de Releases en GitHub.
- **Acción**: Modificar `nx.json` para agregar `"createRelease": "github"` en la configuración de `projectChangelogs`.
- **Por qué**: Esto permite que Nx cree automáticamente el Release en GitHub con el changelog generado y los tags correspondientes (ej. `paquete@1.0.0`) cuando se ejecute el comando.

## 2. Creación del Workflow de CD (`.github/workflows/cd.yml`)
Crearemos un nuevo workflow que se active **solo** cuando haya un push a `master` (usualmente después de un merge).

### Pasos del Workflow:
1.  **Checkout & Setup**: Clona el repo e instala pnpm/node.
2.  **Configuración de Git**: Configura el usuario `github-actions[bot]` para poder hacer commits (version bump + changelog).
3.  **Nx Release Version**:
    *   Analiza los commits (Conventional Commits).
    *   Determina la nueva versión.
    *   Actualiza `package.json`.
    *   Genera `CHANGELOG.md`.
    *   **Crea un commit y tag** en git directamente en `master`.
4.  **Build**: Compila los paquetes con la nueva versión.
5.  **Nx Release Publish**:
    *   Publica a npm.
    *   Crea el Release en GitHub.
    *   Usa `NPM_TOKEN` (secreto) para autenticación (bypasseando OTP).

## 3. Requisitos para el Usuario (Tú)
Necesitarás realizar estas configuraciones en GitHub una vez que yo termine:
1.  **NPM_TOKEN**: Crear un "Automation Token" en npmjs.com y agregarlo como secreto en el repo (`NPM_TOKEN`). Este token salta el requisito de 2FA/OTP en CI.
2.  **Permisos**: Asegurar que los Actions tengan permisos de escritura (Read and write permissions) en el repositorio.

## Respuesta a tus dudas específicas:
*   **¿PR para Changelog?**: No es necesario ni recomendado para CD puro. Nx Release hace el commit del changelog y el tag automáticamente en el mismo paso. Crear una PR intermedia rompe la automatización total.
*   **¿OTP?**: En local usas OTP porque es una sesión interactiva. En CI, usamos un **Automation Token** de npm que no requiere OTP.
